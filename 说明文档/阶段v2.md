# AgentOS V2 开发阶段详解

> **周期**：第 5–8 周（单人全职）
> **前置依赖**：V1 已完成并验证（用户认证、Agent CRUD、对话闭环、Runtime 基础架构）
> **核心目标**：集成外部工具和记忆系统，让 Agent "能调用工具、能记住上下文"
> **产出**：Agent 可调用搜索/文件操作等工具，具备短期/长期记忆能力，支持简单 RAG 知识库查询

---

## 1. 技术选型补充

V2 在 V1 技术栈基础上新增以下组件：

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| MCP 协议 | fastmcp | 2.x | 统一工具调用协议（Model Context Protocol） |
| 向量数据库 | Qdrant | 1.9+ | 语义向量存储（记忆检索 + RAG） |
| 嵌入模型 | fastembed | 0.3+ | 文本向量化（CPU 友好，Qdrant 同生态） |
| Python HTTPX | httpx | 0.27+ | 异步 HTTP 调用（MCP SSE 传输 + 搜索 API） |
| **后端新增** | | | |
| Jackson JSON | 已有 | - | 工具 Schema 的 JSONB 序列化 |
| **运行时新增** | | | |
| qdrant_client | 1.9+ | Qdrant 向量数据库客户端 |
| fastmcp | 2.x | MCP 客户端/服务端 |
| httpx | 0.27+ | 异步 HTTP |

> **向量数据库选择说明**：选用 Qdrant 而非 Milvus。Qdrant 单二进制即可运行，与 fastembed 同属 Qdrant 生态，对 solo 开发运维友好；Milvus 需 etcd + MinIO 等多组件编排，运维成本高，V3 引入向量检索时再部署。
>
> **fastmcp 版本**：2.x 尚在快速迭代，实施时锁定具体次版本号（如 `fastmcp==2.0.1`），避免升级断裂。
>
> **后端 HTTP 客户端**：文档中示例使用 RestTemplate，SpringBoot 3.x 已将其标记为维护模式。实施时建议迁移至 `RestClient` 或 `WebClient`，尤其涉及流式响应时 WebClient 更合适。

```sql
 -- V2__tool_and_memory.sql

  -- ============================================================
  -- 工具中心
  -- ============================================================

  -- MCP 服务注册表：每个 MCP 服务器对应一条记录
  CREATE TABLE mcp_server (
      id          BIGSERIAL    PRIMARY KEY,
      name        VARCHAR(100) NOT NULL UNIQUE,
      description VARCHAR(255),
      endpoint    TEXT         NOT NULL,                -- TEXT 避免 URL 被截断
      transport   VARCHAR(20)  DEFAULT 'sse',           -- sse / stdio
      status      SMALLINT     DEFAULT 1,               -- 1:启用 0:禁用
      created_at  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
      updated_at  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
  );

  -- 工具表：MCP 提供或 Runtime 内置的工具
  CREATE TABLE tool (
      id            BIGSERIAL    PRIMARY KEY,
      mcp_server_id BIGINT       REFERENCES mcp_server(id) ON DELETE CASCADE,  -- NULL = 内置工具
      name          VARCHAR(100) NOT NULL,
      description   TEXT,
      schema        JSONB,                                -- OpenAI function calling schema
      source        VARCHAR(20)  DEFAULT 'builtin',        -- builtin / mcp
      status        SMALLINT     DEFAULT 1,
      version_hash  VARCHAR(64),                           -- schema 哈希，判断是否变更
      created_at    TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
      synced_at     TIMESTAMP                              -- 最近一次从 MCP 同步的时间
  );

  -- Agent 与工具的关联表（只有 mcp 工具需要显式绑定）
  CREATE TABLE agent_tool (
      id          BIGSERIAL    PRIMARY KEY,
      agent_id    BIGINT       NOT NULL REFERENCES agent(id) ON DELETE CASCADE,
      tool_id     BIGINT       NOT NULL REFERENCES tool(id) ON DELETE CASCADE,
      created_at  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
      UNIQUE(agent_id, tool_id)
  );

  -- ============================================================
  -- 记忆中心
  -- ============================================================

  -- 知识库文档（RAG 数据源）
  CREATE TABLE knowledge_doc (
      id               BIGSERIAL    PRIMARY KEY,
      agent_id         BIGINT       REFERENCES agent(id) ON DELETE CASCADE,  -- NULL = 全局
      title            VARCHAR(200),
      content          TEXT         NOT NULL,
      source           VARCHAR(50)  DEFAULT 'manual',       -- manual / upload / web
      chunk_count      INT          DEFAULT 0,
      embedding_status SMALLINT     DEFAULT 0,               -- 0:未索引 1:索引中 2:已索引
      created_at       TIMESTAMP    DEFAULT CURRENT_TIMESTAMP,
      updated_at       TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
  );

  -- 记忆记录（长期记忆，从对话中提取的重要信息）
  CREATE TABLE agent_memory (
      id          BIGSERIAL    PRIMARY KEY,
      agent_id    BIGINT       NOT NULL REFERENCES agent(id) ON DELETE CASCADE,
      user_id     BIGINT       NOT NULL REFERENCES sys_user(id) ON DELETE CASCADE,
      memory_type VARCHAR(20)  DEFAULT 'episodic',          -- working / episodic / semantic
      content     TEXT         NOT NULL,
      importance  FLOAT        DEFAULT 0.5,
      metadata    JSONB,
      created_at  TIMESTAMP    DEFAULT CURRENT_TIMESTAMP
  );

  -- 索引
  -- ============================================================

  CREATE INDEX idx_tool_source             ON tool(source);
  CREATE INDEX idx_agent_tool_agent_id     ON agent_tool(agent_id);
  CREATE INDEX idx_agent_tool_tool_id      ON agent_tool(tool_id);
  CREATE INDEX idx_knowledge_doc_agent_id  ON knowledge_doc(agent_id);

  -- agent_memory 常用查询的复合索引
  CREATE INDEX idx_memory_lookup           ON agent_memory(agent_id, memory_type, importance DESC);
  CREATE INDEX idx_memory_created          ON agent_memory(agent_id, created_at DESC);
```

#### 工具同步流程（仅 MCP 工具）

> 内置工具（builtin）由 Runtime 在启动时通过 `ToolRegistry.register_builtins()` 自动注册，无需同步。

MCP 工具同步流程：

```
1. 用户注册 MCP 服务（提供 endpoint URL）
2. 后端调用 Runtime: POST /runtime/mcp/sync
   Body: { endpoint: "http://xxx:8000/mcp" }
3. Runtime 的 MCPClient 连接该服务器
4. 调用 list_tools() 获取所有工具的 schema
5. 返回 tools 列表（source="mcp"）
6. 后端保存到 tool 表（计算 version_hash 存入）
```

```java
// MCP 同步工具
@Component
public class McpSyncService {

    private final RestTemplate restTemplate;

    @Value("${runtime.base-url}")
    private String runtimeBaseUrl;

    /**
     * 让 Runtime 去连接 MCP 服务器并拉取工具列表。
     */
    public List<ToolSchema> syncTools(String endpoint) {
        String url = runtimeBaseUrl + "/runtime/mcp/sync";
        McpSyncRequest req = new McpSyncRequest(endpoint);
        ResponseEntity<McpSyncResponse> resp = restTemplate.postForEntity(url, req, McpSyncResponse.class);
        return resp.getBody().getTools();
    }
}
```

```java
// ConversationServiceImpl.java — sendMessage 增强
public Message sendMessage(Long conversationId, SendMessageReq req) {
    // 1. 保存 user 消息（同 V1）

    // 2. 获取该 Agent 绑定的 MCP 工具列表（内置工具由 Runtime 自动注册，无需下发）
    List<Tool> tools = agentToolMapper.selectToolsByAgentId(conversation.getAgentId());

    // 3. 将工具 schema 转换为 LLM function calling 格式

            .collect(Collectors.toList());

    req.setMessages(payloads);
    req.setTools(toolSchemas);  // V2 新增：MCP 工具 schema

    // 5. Runtime 返回结果可能包含工具调用中间结果
    AgentRuntimeClient.InvokeResponse resp = agentRuntimeClient.invoke(...);
}
```

```python
"""
ToolRegistry —— 管理 Agent 可用的所有工具。

工具来源分两类：
  1. 内置工具（builtin）：Runtime 启动时通过 register_builtins() 自动注册，

  1. 启动时自动注册 CalculatorTool、SearchTool 等内置工具
  2. MCP 工具由后端同步到 tool 表后，在 Agent 启动时推送到 Runtime

class ToolRegistry:

    def register_mcp_tools(self, tools: List[Dict], client: MCPToolClient): ...
    async def execute(self, name: str, params: Dict) -> str: ...
    def get_openai_schemas(self) -> List[Dict]: ...
```

### 5.3 ReAct Agent

```python
"""
ReActAgent —— 推理-行动循环 Agent（V2 核心新增）。

流程：
  Thought → Action → Observation → Thought → ... → Finish

与 V1 SimpleAgent 的区别：
  - SimpleAgent: 单次 LLM 调用，直接返回
  - ReActAgent: 多轮循环，每轮可调用工具，直到输出 Finish

与 SimpleAgent 的关系：
  - SimpleAgent 保留，用于不需要工具的纯对话场景
  - ReActAgent 继承 BaseAgent，增加 ToolRegistry 和循环逻辑
"""
class ReActAgent(BaseAgent):
    def __init__(
        self,
        name: str,
        llm_client: LLMClient,
        system_prompt: str = "",
        tool_registry: Optional[ToolRegistry] = None,
        max_steps: int = 5,
    ):
        ...

    async def run(self, input_text: str, **kwargs) -> str:
        """
        ReAct 循环：
          1. 构建 prompt（含工具描述）
          2. 调用 LLM
          3. 解析输出（Thought / Action）
          4. Action=Finish → 返回最终答案
          5. Action=Tool → 执行工具 → Observation → 回到 2
          6. 达到 max_steps → 强制结束
        """
        ...
```

### 5.4 LLMClient 增强

V2 的 `LLMClient.invoke()` 需要支持 tools 参数（OpenAI function calling）：

```python
# runtime/core/llm.py — 增强
class LLMClient:
    async def invoke(
        self,
        messages: List[Dict],
        system_prompt: Optional[str] = None,
        tools: Optional[List[Dict]] = None,  # V2 新增
    ) -> Dict:
        """
        调用 LLM。

        V2 新增 tools 参数：
          - 传入 OpenAI function calling schema 列表
          - LLM 可选择返回 tool_calls 而非普通 content
          - ReActAgent 根据 tool_calls 执行工具
        """
        kwargs = {
            "model": self.model_name,
            "messages": full_messages,
            "temperature": self.temperature,
            "max_tokens": self.max_tokens,
        }
        if tools:
            kwargs["tools"] = tools  # OpenAI function calling

        resp = await self.client.chat.completions.create(**kwargs)
        ...
```

### 5.5 MCP 同步 API

```python
# runtime/api/mcp_routes.py
"""
MCP 工具同步端点。

后端注册 MCP 服务后，调用此接口让 Runtime 连接 MCP 服务器并拉取工具列表。
"""
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from tools.mcp_client import MCPToolClient

router = APIRouter()

class McpSyncRequest(BaseModel):
    endpoint: str
    transport: str = "sse"

@router.post("/mcp/sync")
async def sync_mcp_tools(req: McpSyncRequest):
    """连接 MCP 服务器，拉取工具列表和 schema。"""
    try:
        client = MCPToolClient(endpoint=req.endpoint, transport=req.transport)
        tools = await client.list_tools()
        return {"status": "ok", "tools": tools}
    except Exception as e:
        raise HTTPException(status_code=502, detail=f"MCP 连接失败: {e}")
```

## 7. 向量检索集成规划（V2 可选，V3 必选）

V2 采取渐进策略：

| 阶段 | 检索方式 | 存储 | 适用场景 |
|------|---------|------|---------|
| V2 MVP | PostgreSQL 关键词匹配（ILIKE + tsvector） | PostgreSQL | 文档量 < 1000 条 |
| V2.5 | 嵌入 + PostgreSQL 存储向量 | pgvector 插件 | 文档量 < 10000 条 |
| V3 | Qdrant 向量检索 | Qdrant 1.9+ | 大规模语义检索 |

V2 默认使用简单关键词匹配，V3 引入 Qdrant 做语义检索。选用 Qdrant 而非 Milvus，原因：
- **部署简单**：单二进制启动，无需 etcd/MinIO 等组件
- **同生态**：fastembed 由 Qdrant 团队维护，Python 客户端集成顺畅
- **够用**：单机模式可支撑百万级向量，满足 AgentOS 的规模预期

## 11. 设计原则与取舍

1. **工具优先，记忆次之** — 工具调用是 V2 的核心价值，记忆系统做 MVP 版本
2. **内置工具直调，MCP 管外部** — 内置工具（Calculator、Search）由 Runtime 本地注册直接调用；外部工具通过 MCP 协议接入。tools 表用 `source` 字段区分两类工具，`mcp_server_id` 为空即为内置工具
3. **ReAct 作为默认 Agent 策略** — V2 默认 Agent 从 SimpleAgent 升级为 ReActAgent（无工具时退化为基础对话）
4. **PostgreSQL 兜底** — 在引入 Qdrant 前，所有数据存在 PostgreSQL（降低运维成本）
5. **不做的**：
   - 流式输出工具调用过程（V3）
   - 可视化工作流编排（V3）
   - 多 Agent 团队协作（V3）
   - Qdrant 向量检索（V3）
