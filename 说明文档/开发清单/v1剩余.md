  协作要点

  1. API 约定：两位开发都依赖 frontend/src/api/ 目录，可以先定好接口层，A 写管理端 API，B 写聊天 API
  2. 路由合并：A 先搭好 router/index.ts 骨架，B 在已有路由上加 chat 页
  3. 联调窗口：B4 集成联调需要后端和运行时已启动，建议后端开发者（你）在 B 开始前端工作前确保 backend + runtime 可运行
  4. 前后端并行开发：前端开发时可用 mock 数据，后端 API 已就绪后切到真实调用
