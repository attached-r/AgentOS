# AgentOS V1 开发阶段详解

> **周期**：第 1–4 周（单人全职）  
> **目标**：搭建完整项目骨架，实现单智能体注册、对话、任务执行的闭环，跑通"用户发任务 → Agent 执行 → 结果返回"的最小链路。  
> **产出**：可演示的 Web 应用，用户能登录、创建 Agent、发起对话/任务、查看结果。

### 4.2 模块划分与实现顺序

#### 4.2.1 common 模块 — 统一返回与异常

**类**：`R<T>`（统一响应体）、`BusinessException`（业务异常）、`GlobalExceptionHandler`（全局异常拦截）

```java
// R.java — 统一响应
@Data
@Accessors(chain = true)
public class R<T> {
    private int code;        // 200 成功，非 200 失败
    private String msg;
    private T data;

    public static <T> R<T> ok(T data) {
        return new R<T>().setCode(200).setMsg("success").setData(data);
    }
    public static <T> R<T> fail(int code, String msg) {
        return new R<T>().setCode(code).setMsg(msg);
    }
}
```

**为什么**：统一前端解析逻辑，错误时前端 Axios 拦截器统一弹提示。

#### 4.2.2 config 模块 — 安全与跨域

**核心类**：
- `SecurityConfig`：Spring Security 配置，放行 `/api/auth/**`，其余请求需 JWT
- `JwtAuthFilter`：OncePerRequestFilter，从 Header 提取 Token 并校验
- `JwtUtil`：Token 生成/解析工具
- `CorsConfig`：跨域配置（Vite dev server 默认端口 5173 → 后端 8080）

**JWT 流程**：
```
登录 → 后端校验账号密码 → 颁发 access_token(1h) + refresh_token(7d)
前端存 localStorage → Axios 拦截器自动附 Authorization Header
access_token 过期 → 用 refresh_token 换新 → 透明续期
```

#### 4.2.5 对话 / 消息 API

| API | 说明 |
|-----|------|
| `POST /api/conversations` | 创建新会话，关联 agent_id |
| `GET /api/conversations` | 当前用户的会话列表 |
| `GET /api/conversations/{id}/messages` | 获取某会话的全部消息 |
| `POST /api/conversations/{id}/messages` | **发送消息并触发 Agent 执行** |

**消息发送流程（核心闭环）**：
```
1. 前端 POST /api/conversations/{id}/messages { content: "..." }
2. 后端收到请求，保存 user 消息到 message 表
3. 后端通过 HTTP 调用 Python Runtime: POST /runtime/agents/{agentId}/invoke
   Body: { conversation_id, messages: [...] }
4. Python Runtime 调用 LLM → 得到回复
5. Python Runtime 返回回复文本 + token 用量
6. 后端保存 assistant 消息到 message 表
7. 后端将 assistant 消息返回前端

（V1 为简化，用同步 HTTP；V2 改为异步消息队列）
```

### 5.2 Agent 执行器（核心逻辑）

```python
# runtime/agents/executor.py
from openai import OpenAI

class AgentExecutor:
    """单智能体执行器 — V1 只做最简单的调用链：user msg → LLM → response"""

    def __init__(self, agent_config: dict):
        self.client = OpenAI(
            base_url=agent_config.get("base_url", "https://api.openai.com/v1"),
            api_key=agent_config.get("api_key"),
        )
        self.system_prompt = agent_config["system_prompt"]
        self.model = agent_config.get("model_name", "gpt-4o-mini")
        self.temperature = agent_config.get("temperature", 0.7)
        self.max_tokens = agent_config.get("max_tokens", 4096)

    async def invoke(self, messages: list[dict]) -> dict:
        """调用 LLM，返回回复内容和用量"""
        full_messages = [{"role": "system", "content": self.system_prompt}] + messages
        resp = self.client.chat.completions.create(
            model=self.model,
            messages=full_messages,
            temperature=self.temperature,
            max_tokens=self.max_tokens,
        )
        return {
            "content": resp.choices[0].message.content,
            "usage": {
                "prompt_tokens": resp.usage.prompt_tokens,
                "completion_tokens": resp.usage.completion_tokens,
                "total_tokens": resp.usage.total_tokens,
            },
        }
```

### 5.3 Invoke 接口

```python
# runtime/workflows/single_agent.py
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel
from agents.executor import AgentExecutor
from agents.registry import agent_registry  # 内存字典

router = APIRouter()

class InvokeRequest(BaseModel):
    conversation_id: str
    messages: list[dict]  # [{"role": "user", "content": "..."}]

@router.post("/{agent_id}/invoke")
async def invoke_agent(agent_id: int, req: InvokeRequest):
    config = agent_registry.get(agent_id)
    if not config:
        raise HTTPException(status_code=404, detail="Agent not found")
    executor = AgentExecutor(config)
    result = await executor.invoke(req.messages)
    return result
```

// ChatView.vue 核心逻辑
const messages = ref<Message[]>([])
const loading = ref(false)

async function sendMessage() {
  if (!inputText.value.trim() || loading.value) return

  // 1. 追加用户消息到本地（立即显示）
  messages.value.push({ role: 'user', content: inputText.value })
  loading.value = true

  try {
    // 2. 调后端 API
    const res = await api.sendMessage(conversationId, inputText.value)
    // 3. 追加 AI 回复
    messages.value.push({ role: 'assistant', content: res.data.content })
  } catch (e) {
    // 4. 失败处理
    ElMessage.error('发送失败')
  } finally {
    loading.value = false
  }
}

## 7. 后端 ↔ 运行时联调（第 4 周前 2 天）

### 7.1 完整调用链路

```
浏览器 → SpringBoot API        → Python FastAPI        → OpenAI API
         POST /api/conversations  POST /runtime/agents/   chat.completions
           /{id}/messages           {id}/invoke             .create
         ← 保存 message 表        ← 返回 content+usage   ← stream response
         ← 返回前端
```

### 7.2 后端调用 Python Runtime 的 HTTP 客户端

```java
// runtime/AgentRuntimeClient.java
@Component
public class AgentRuntimeClient {

    @Value("${runtime.base-url:http://localhost:8001}")
    private String runtimeBaseUrl;

    private final RestTemplate restTemplate = new RestTemplate();

    public InvokeResult invokeAgent(Long agentId, String conversationId,
                                    List<MessageDTO> messages) {
        String url = runtimeBaseUrl + "/runtime/agents/" + agentId + "/invoke";
        InvokeRequest req = new InvokeRequest(conversationId, messages);
        ResponseEntity<InvokeResult> resp = restTemplate.postForEntity(url, req, InvokeResult.class);
        return resp.getBody();
    }
}
```

### 7.3 错误处理策略

| 场景 | 后端行为 | 前端表现 |
|------|---------|---------|
| Python Runtime 不可用 | 返回 503，记录 task_log | 提示"智能体服务暂不可用" |
| LLM API 超时 | 返回 504，记录 error log | 显示"请求超时，请重试" |
| LLM 返回格式异常 | 透传原始内容 | 正常显示（V1 不处理重试） |
| Token 耗尽 | 返回具体错误信息 | 提示"API 额度不足" |

## 11. 常见问题与避坑

1. **JWT 密钥泄露**：生产环境从环境变量读取，不硬编码在配置文件
2. **Python 运行时配置同步**：V1 用启动时一次性推送，如果运行时重启需手动触发同步；可加一个 `/runtime/agents/sync` 接口在管理后台提供"同步"按钮
3. **CORS 跨域**：Vite 开发端口 5173，后端 8080，务必在 SecurityConfig 中配置 CORS
4. **对话上下文长度**：V1 不做滑动窗口，超过模型上下文长度会报错，简单处理：捕获后提示用户"对话过长，请开启新会话"
5. **API Key 管理**：Python 运行时的 LLM API Key 通过环境变量注入，不存数据库
